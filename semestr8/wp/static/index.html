Задание: 1. HTTP-сервер. Простой веб-сервер, предоставляющий доступ к статическим ресурсам.

Требования к сайту:

  • Страница с информацией о разработчике (т.е. о Вас), размеченная с использованием микроформата hCard.
  • Карта сайта в формате SiteMap. http://wp.rfedorov.ru/sitemap.xml
  • В блоке head каждой страницы – тег link на ваш профиль в Google
  • RSS в формате RSS 2.0 или Atom.
  • На каждой странице любой счетчик посещений с открытой статистикой (например, http://liveinternet.ru).
  • Прямая ссылка со стартовой страницы на сайт кафедры АСОИУ.
  • Средства интеграции с социальными веб-сервисами (см. API mail.ru, yandex.ru, odnoklassniki.ru, vkontakte.ru и т.д.).

Требования к оформлению:

  • Блочная («div-ная») верстка
  • Кросс-браузерность.
  • Подержка CSS (минимум – CSS 2) для различных типов медиа (screen, print, handheld).
  • Валидный HTML и CSS.
  • Обязательное наличие атрибутов title у ссылок и alt у изображений.
  • Относительные адреса у внутренних ссылок и абсолютные – у внешних.
  • ВНИМАНИЕ! В качестве основы для макета сайта (общей идеи оформления, концепции дизайна) разрешается использовать тематические шаблоны для сайтов, распространяемые под лицензией Creative Commons или подобной.

Краткое описание:

В ходе работы был реализован собственный веб сервер rfedorov, используя сторонний соккет сервер на языке Python. Предоставлен валидный HTML, блочная верстка и CSS совместимы с большинсвом современных браузеров. Данный сайт имеет как динамические, так и статические страницы. На хостинг сервере используется несколько веб серверов для работы сайта.

Хостинг и настройки:

  • Используется доменное имя третьего уровня wp.rfedorov.ru
  • Данный сайт выложен на сторонний облачный сервер DigitalOcean
  • Для работы этого сайта используется собственный веб сервер rfedorov с портом 8047
  • Используется прозрачный прокси nginx с портом 80

Возможности веб сервера:

  • Отображение индексного файла index.html при отстутствии пути
  • Отображение css файлов
  • Отображение js файлов
  • Отображение ошибок при неправильных запросах
  • Поддержка keep-alive
  • Автоматическое добавление "правильных" заголовков в зависимости от типа файла
  • Динамическая страница /dynamic/, которая отображает серверное время

Настройки прозрачного прокси nginx:

На хостинг сервере используется несколько веб серверов. Собственный веб сервер rfedorov c портом 8047. Веб сервер nginx c портом 80, который прозрачно проксирует все запросы вида .wp.rfedorov.ru с порта 80 на 8047. Это сделано для того, чтобы остальные сайты на 80 порту данного сервера продолжали корректно работать. Настройки nginx: 
server {
    server_name .wp.rfedorov.ru;
    error_log /var/log/nginx/wp-error.log;

    location / {
        proxy_pass http://127.0.0.1:8047;
        proxy_pass_header Server;
        proxy_set_header Host $http_host;
        proxy_redirect off;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Scheme $scheme;
    }
}
